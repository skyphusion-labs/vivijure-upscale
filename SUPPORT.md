# Getting help

Thanks for using the Vivijure video upscale module. Here is where to go when you need a hand.

## Where to ask

1. **This repo's [GitHub Issues](../../issues) first.** Bug reports, questions, and feature ideas are
   all welcome. Please search the open and closed issues before you file one, in case it is already
   answered.
2. **Email support@skyphusion.org next.** For anything that does not fit a public issue, the crew reads
   this mailbox.

New module idea for Vivijure? Those stay `help wanted` so the community can build them; open one on the
main [vivijure](https://github.com/skyphusion-labs/vivijure) repo.

## What to include

Give us enough to reproduce the problem in one pass:

- The **version** you are running (release tag or commit SHA).
- What you **expected**, and what **actually happened**.
- **Steps to reproduce**, with a minimal example if you can.
- Relevant **logs** or error output. Scrub anything secret first (tokens, keys, private URLs).

## What to expect

This is free, open-source software, supported by the maintainers on a **best-effort** basis. There is
no paid support tier and no guaranteed response time; we get to issues as we can, and clear,
reproducible reports get help fastest.

**Security problem?** Do not open a public issue. See [SECURITY.md](SECURITY.md) and report it
privately to security@skyphusion.org.
